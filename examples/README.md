# Getting started

Here, we provide examples for the folllowing topics:
- creating NeXus application definitions
- convert your data (raw data, numerical data, metadata) into a NeXus/HDF5 file using the [built-in readers of pynxtools](https://fairmat-nfdi.github.io/pynxtools/reference/built-in-readers.html).
- creating your own reader plugin for pynxtools

## Dataconverter and readers

In the documentation, you can learn more about the `pynxtools`
[dataconverter](../src/pynxtools/dataconverter/README.md) and its reader implementation:

- [Learn > ... > Data conversion in pynxtools](https://fairmat-nfdi.github.io/pynxtools/learn/pynxtools/dataconverter-and-readers.html) — learn how the data converter works 
- [Learn > ... > Build your own pynxtools plugin](https://fairmat-nfdi.github.io/pynxtools/how-tos/pynxtools/build-a-plugin.html) — how-to build a reader plugin for pynxtools,
- [Reference > Built-in pynxtools readers](https://fairmat-nfdi.github.io/pynxtools/reference/built-in-readers.html) — built-in readers in the core `pynxtools`
- [Reference > FAIRmat-supported pynxtools plugins](https://fairmat-nfdi.github.io/pynxtools/reference/plugins.html) — existing `pynxtools` reader plugins

For giving feedback to specific parsers/readers/data extractors, please checkout the domain-specific `pynxtools` plugins and their examples or contact the respective developers directly.
